# Подготовка виртуальной машины

## Склонируйте репозиторий

Сначала копируем себе оригинальный репозиторий в личное пространство Github в качестве шаблона (через UI Github)
```
# оригинал
https://github.com/yandex-praktikum/mle-project-sprint-4-v001.git
```

Далее настраиваем рабочее пространство на ВМ
```
# клонируем репозиторий
git clone https://github.com/khromenko/mle-project-sprint-4-v001.git
```

## Активируйте виртуальное окружение

Используйте то же самое виртуальное окружение, что и созданное для работы с уроками. Если его не существует, то его следует создать.

Создать новое виртуальное окружение можно командой:

```
python3 -m venv .venv_mle-project-sprint-4-v001
```

После его инициализации следующей командой

```
. .venv_mle-project-sprint-4-v001/bin/activate
```

установите в него необходимые Python-пакеты следующей командой

```
pip install -r requirements.txt
```

# Расчёт рекомендаций

Код для выполнения первой части проекта находится в файле `recommendations.ipynb`. Изначально, это шаблон. Используйте его для выполнения первой части проекта.

# Сервис рекомендаций

- Код сервиса рекомендаций находится в файле `app\recommendations_service.py`.
- Настройки логирования находится в файле `app\logging_config.py`.
- Поддерживается 3 режима работы рекомендаций - оффлайн, онлайн и полный (онлайн + оффлайн)

## Алгоритм работы основного сервиса рекомендаций
### Оффлайн рекомендации
- По запросу для пользователя (`user_id`) возвращаются предрасчитанные ml-моделью -рекомендации.
- За взаимодействие с ml-моделью отвечает класс ModelHandler (`app/model_handler.py`)
- Если для пользователя не найдены персональные рекомендации, то будут возвращены обезличенные "общие рекомендации" - топ-популярных
    - такое событие не является штатным, поэтому ведется учет статистики таких событий для возможности дальнейшего анализа причин
- Перед запуском необходимо прописать пути к подготовленным на Этапе 3 файлам модели в .env файле
    - ML_MODEL_USER_DATA_PATH = data/recsys/recommendations.parquet
    - ML_MODEL_COMMON_DATA_PATH = data/recsys/top_popular.parquet
    - ML_MODEL_SIM_DATA_PATH = data/recsys/similar.parquet

### Онлайн рекомендации
- По запросу для пользователя (`user_id`) сначала происходит поиск свежей истории взаимодействия через обращение к отдельному сервису - Event store service (FastApi приложение - `app/events/user_event_service.py`)
- Если Event store service вернул набор объектов (список item id), то по каждому объекту запрашивается список похожих на него через обращение к отдельному сервису - Content service (FastApi приложение `app/content/sim_model_handler.py`)
- полученные списки похожих объектов объединяются в один большой список, сортируются по параметру `score` каждого похожего объекта и отсекается топ-n самых значимых (похожих) объектов 
    - получившийся список возвращается как результирующий набор онлайн рекомендаций

### Онлайн + оффлайн рекомендации ("полный" режим)
- По запросу для пользователя (`user_id`) происходит формирование двух списков рекомендаций
    - список 1 - онлайн рекомендации 
    - список 2 - офлайн рекомендации 
    - алгоритм формирование обоих списков описан выше
- Самые выжные объекты расположены в начале списков
- Полученные списки 1 и 2 объединяются (смешиваются) в один общий по схеме парного чередования
    - в цикле по i от 0 до тех пор, пока есть элементы в списке 1 или 2
        - добавляется очередной i-ый элемент из списка 1
        - добавляется очередной i-ый элемент из списка 2
        - если элементы в списке закончилисись, то добавляются элементы из того списка, где еше есть элементы
- Полученный смешенный список отрезается до топ-n и возвращается как результат работы алгоритма


# Запуск сервисов

Все команды выполняются из корневой директории проекта (и активированной venv).

- Запуск основного сервиса рекомендаций:

    ``` bash
    uvicorn app.recommendations_service:app --port 8000
    ```

- Запуск сервиса сбора событий пользователя (Event store service)

    ``` bash
    uvicorn app.events.user_event_service:app --port 8001
    ```

- Запуск сервиса управления контентом для получения похожих объектов (Content service)

    ``` bash
    uvicorn app.content.content_service:app --port 8002
    ```

- Все приведенные ниже команды выполняются из корневой директории проекта (и активированной venv)

## Тестирование основного сервиса рекомендаций

- Код для тестирования основного сервиса находится в файле `test\test_service.py`.
- Доступен вызов 3-х тест-кейсов через параметр запуска модуля (--case)
    - online - для получения онлайн рекомендаций по истории без персональных рекомендации
    - offline - для получения оффлайн персональных рекомендаций (без онлайн истории)    
    - full - для получения одновременно онлайн и оффлайн рекомендаций

- Перед запуском необходимо сформировать онлайн историю взаимодействия, с помощью вызова модуля тестирования `test_event_store.py` - будут сформированы случайные 20 записей с историей для 10 пользователей

    ```bash
    python -m pytest test/test_event_store.py -s
    ```
- Затем необходимо поочереди запустить модуль тестирования в 3-х разных режимах

    ```bash
    python -m pytest test/test_service.py --case=online -s
    python -m pytest test/test_service.py --case=offline -s
    python -m pytest test/test_service.py --case=full -s
    ```

- результаты выполнения будут залогированы в директории `logs`
    - app.log - содержит вывод из всех логеров
    - test_service.log - содержит вывод модуля тестирования `test_service.py`
    - прочие файлы <module_name>.log - соответсвенно содержат вывод логов конкретных модулей

- в логах не должно быть ошибок (ERROR)
